## 3. 社区痛点（按呼声排序）

1. **Backlink 破损（HIGH）**——改标题/编号后 `[[file#heading]]` 失效；只有 Header Enhancer 解决。
   Number Headings Issue #26 长期未决。**这是我们唯一的硬短板，列为 1.0 前置必做。**
2. **配置散落（MEDIUM）**——竞品全靠逐文件 frontmatter，无 vault 级统一配置。
   **我们的模板 + 路径规则正是答案**，是最大差异化。
3. **性能/卡顿（MEDIUM）**——老牌插件 TOC、分隔符空格触发卡顿。我们有 fuzz 兜底。
4. **视觉缩进 + 编号合一（MEDIUM）**——仅 Auto Heading 同时具备。我们暂不跟进。
5. **混合样式（LOW）**——小众，我们已部分覆盖（祖先统一阿拉伯）。

## 4. 定位结论

- **目标用户分层**：
  - **轻度（80%）**：装上即用默认模板（`/` 根规则 → `1.1.1`），**零配置**——开箱体验对齐 Number Headings。
  - **进阶（20%）**：多文档风格 / 中文书排版 / 导入旧编号清理——**模板 + 路径规则 + 清除/清理**，竞品全无。
  - 「开箱即用的简单」与「需要时的强大」由**优先级最低的默认规则**统一，不互相打架——这是回应「过度设计」质疑的核心。
- **不跟进**：Visual-only 模式、视觉缩进（Auto Heading 强项，与我们「burn-in 写入」哲学相悖，留观察）。
- **必补**：**Backlink 同步**——补齐后我们在「灵活性 + 可靠性 + backlink」三轴同时领先，再无明显短板。

## 5. 发布策略

| 项 | 决策 | 理由 |
|----|------|------|
| **Backlink 同步** | **1.0 发布前置必做** | 唯一硬短板、社区呼声第一；上架前补齐胜过事后追赶 |
| **版本号** | M7 完成即发 **1.0.0** | 脱离 `0.M.*` 内部打磨号；消除「0.x = 不稳定」心理（对标 NH v1.16） |
| **README** | 英文优先 + 突出模板/路径/清理 | Obsidian 主社区是英文；差异化点要第一眼可见 |
| **时机** | 趁 Number Headings 停更窗口 | 龙头 2.5 年未更、issue 堆积，是切入良机 |

> 详细 Roadmap 与 Backlink 设计见 [`spec.md` §5 / §3.12](./spec.md)。
