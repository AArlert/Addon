# 场景 04：白名单结构性标题（⚠ 功能开发中 / Milestone 4）

> **当前状态**：白名单匹配（Milestone 4）**尚未接入插件主流程**——所以现在复制下面的内容实测，
> 「目录 / 参考文献」等标题**仍会被正常编号**。本文件先把**目标行为**记录下来，待 M4 上线后即可用它验收。

设计目标：命中白名单的结构性标题（如 `目录`、`附录`、`参考文献`）**不写编号前缀，且不占用计数器槽位**
（计数器既不累加也不归零，因此不产生跳号）。

**M4 上线后的预期结果**：

- `## 目录` → 不编号（命中白名单）
- `## 正文` → `1`（`目录` 未占槽位，故正文仍是 1 而非 2）
- `### 小节` → `1.1`
- `## 参考文献` → 不编号（命中白名单）

**当前（M4 未上线）的实际结果**：上述四个 H2/H3 都会被编号——`目录`=1、`正文`=2、`小节`=2.1、`参考文献`=3。

**可复制内容**：

```md
# 带结构性标题的文档

## 目录

## 正文

### 小节

## 参考文献
```
