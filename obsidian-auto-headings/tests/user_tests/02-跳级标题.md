# 场景 02：跳级标题（缺失中间级以 `0` 占位）

当标题层级跳跃（如 H3 后直接跟 H5，中间没有 H4），插件**仍让序号段数等于标题深度**
（H_n 有 n−1 段），缺失的中间级别按 `0` 显式占位——既不会把 H5 误当成 H4，也用 `0`
区别于「真实存在的该级标题 = 1」。该级计数器本身仍保持 0，**随后首个真实的该级标题仍从 1 起算**，不被借号。

**默认模板下的预期结果**：

- `## 章` → `1`
- `### 节` → `1.1`
- `##### 细目甲` → `1.1.0.1`（H5 直接跟在 H3 后，缺失的 H4 显示为 `0`，共四段）
- `##### 细目乙` → `1.1.0.2`
- `#### 子节` → `1.1.1`（首个真实 H4，从 1 起算，未被前面跳级借号）
- `###### 末级` → `1.1.1.0.1`（H6 跟在 H4 后，缺失的 H5 显示为 `0`，共五段）

对照：若中间级别**真实存在**，则不会出现 `0`——例如先有 `#### 子节`(`1.1.1`)，其下的
`##### 细目` 就是 `1.1.1.1` 而非 `1.1.0.1`。

**可复制内容**：

```md
# 跳级测试文档

## 章

### 节

##### 细目甲

##### 细目乙

#### 子节

###### 末级
```
