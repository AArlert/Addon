# 场景 03：多个 H1（错位 H1 的处理）

一篇文档里第一个 H1 视为「文档标题」（不编号、不降级）。此后再出现的 H1 属**错位 H1**，
处理方式取决于触发模式：

- **实时编辑（自动防抖触发）**：错位 H1 仅把本行 `#` 改写为 `##`，**不**触动其下属子树。
- **「立即重新编号」命令（格式化）**：自第二个 H1 起直至文件末尾，所有标题层级**整体下移一级**
  （`#`→`##`、`##`→`###`…，封顶 H6）。

**默认模板下的预期结果**：

实时编辑后：

- `# 第一个标题` → 保持 `#`，不编号
- `## 概述` → `1`
- `# 第二个标题` → 改写为 `## 第二个标题`，编号 `2`
- `## 细节` → 仍是 `##`，编号 `2.1`

执行「立即重新编号」命令后（第二个 H1 及其后整体 +1 级）：

- `# 第一个标题` → 保持 `#`
- `## 概述` → `1`
- `# 第二个标题` → `### 第二个标题`，编号 `1.1`
- `## 细节` → `#### 细节`，编号 `1.1.1`

**可复制内容**：

```md
# 第一个标题（文档标题）

## 概述

# 第二个标题（错位 H1）

## 细节
```
