# 场景 07：改模板配置后再触发，不叠加旧前缀（状态转移）

> 对应 [`doc/testplan.md`](../../doc/testplan.md) **B 类**（B1 / B4 / B5）。这一类是**多步操作**——
> 先用一套配置编号，**再去设置里改一个字段**，回到笔记看下一次自动编号的结果。重点验证：用**旧配置**
> 写出的编号前缀，在改配置后能被**干净替换**，而不是把新前缀叠在旧前缀左边（出现 `一、一 标题` 这种）。

## 为什么单独列这一类

单测「一次编号」往往没问题，bug 几乎都藏在「**已经有编号了、又改配置**」的第二步。每次触发 =
先剥旧前缀、再写新前缀；旧前缀是用**旧分隔符**写的，若剥离器只认**当前**分隔符就会漏剥 → 叠加。

---

## B1：序号样式中文 + 标题间隔符 空格 → 改成「、」

**步骤：**

1. 把当前模板（或默认模板）的起始编号层级对应级别（默认 H2）设为：序号样式 **中文**、标题间隔符 **空格**。
2. 新建笔记，粘贴下面内容，等自动编号 → 应得 `## 一 标题`。
3. **回设置**，把该级标题间隔符从 **空格** 改成 **、**。
4. 回笔记，随便敲一下触发重排。

**预期：** `## 一、标题`（旧的 `一 ` 被替换）。
**不应出现：** `## 一、一 标题`（旧 `一 ` 残留、新 `一、` 叠在左边）。

```md
## 标题
```

---

## B5：标题间隔符「. 」（点+空格）→ 改成 空格

**步骤：** 某级标题间隔符设为 **`. `**（点+空格）→ 编号得 `## 1. 标题` → 改成 **空格** → 再触发。

**预期：** `## 1 标题`。**不应出现：** `## 1 1. 标题`。

```md
## 标题
```

---

## B4：序号间隔符「.」→ 改成「-」

**步骤：** 某级序号间隔符设为 **`.`** → 编号得 `### 1.1 子` → 改成 **`-`** → 再触发。

**预期：** `### 1-1 子`。**不应出现：** `### 1-1 1.1 子`。

```md
## 父
### 子
```

---

## 安全性自查（不该被误伤）

改完上面这些配置后，下面这些**不以序号开头**的标题应当**完全不受影响**（只是正常加/换前缀，
绝不被吃掉文字）：

```md
## API 设计
## 100% 覆盖率
## C++ 入门
```

> 说明：容差剥离只在「标题以序号 token（数字/中文数字/带圈数字）+ 分隔符起头」时才动手；以英文词、
> 符号等起头的标题起手即失配，原样保留。以序号起头的标题（如手写的 `1.1 小节`）会被当作编号前缀
> 覆盖——这与「对编号前缀的手动编辑属预期行为」一致（见 `doc/spec.md` §2.3）。
