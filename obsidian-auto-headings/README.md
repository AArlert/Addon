# obsidian-auto-headings

> 一款 Obsidian 插件，按可完全自定义的模板自动为 Markdown 文件中的标题编号——在你编辑后经防抖延迟把编号前缀直接写入文件，让文档始终整洁一致。**中英双语界面**（自动跟随 Obsidian / 可手动锁定）。

本文件是面向读者的**简介**；详细需求 / 规格 / Roadmap 见 [`doc/spec.md`](./doc/spec.md)，
开发交接日志见 [`doc/log.md`](./doc/log.md)，状态索引见 [`doc/status.jsonl`](./doc/status.jsonl)，
测试场景与真值表（含已知 bug 清单）见 [`doc/testplan.md`](./doc/testplan.md)。
可直接实测的插件产物在 [`release/`](./release/)。

## 当前功能（截至 v0.6.6）

-   **按模板自动编号**：在编辑后防抖触发（延迟可调，默认 300ms），对整个文件重新计算编号前缀并以单一事务写回。
-   **多模板 + 按路径选模板**：可创建多个具名模板；用**路径规则表**把文件夹 / 文件映射到模板
    （`/` 根规则即全局默认），最具体的规则优先、并列取靠后。只对**正在编辑的文件**按其路径解析模板，
    从不全库扫描。
-   **每级格式可定制**：H1–H6 各级可设 `前缀 / 序号样式 / 序号间隔符 / 后缀 / 标题间隔符 / 继承前级`。
    -   序号样式：阿拉伯 `1`、中文 `一`、带圈 `①`、小写字母 `a`、大写字母 `A`、小写罗马 `i`、大写罗马 `I`。
    -   前缀 + 后缀可实现「第 1 章」式编号；关闭「继承前级」可让某级独立编号（如 `a)`、`b)`）。
-   **祖先序号渲染 `ancestorNumeral`**（每模板）：继承前级时祖先段用 `各自样式`（默认，得 `1.a.①`）
    或 `统一阿拉伯`（得 `一` / `1.1` / `1.1.1`，适合中文书：章标题 `一`、子节 `1.1`）。
-   **编号层级区间 `topLevel` / `bottomLevel`**（每模板）：插件**永不改写 `#` 层级**；比 `topLevel` 浅或比
    `bottomLevel`（默认 H6=无下界）深的标题不编号、但仍作计数器重置边界。两者配合可**只编号 H2–H4 这样的
    区间**（0.6.5 新增结束层级）。多个 H1、是否给 H1 编号都由 `topLevel` 决定；收窄区间后降出范围的标题旧
    前缀会自动剥除。
-   **跳级占位策略 `skipFill`**（每模板）：标题跳级（如 H3→H5）时补位 / 不补位可选。
-   **白名单**（每模板）：命中的结构性标题（目录 / 附录 / 参考文献 / 摘要…）不编号、也不占计数器槽位
    （不跳号）。三种匹配方式——`全部`（完全相等）/ `部分`（包含该词）/ `子树`（命中标题及其下属整块豁免）；
    命中判定经归一化（剥行内格式 + NFKC + 折叠空白 + 大小写不敏感）。默认模板预填中英常用结构词表。
-   **中英双语界面**（0.6.5）：设置面板顶部「语言」下拉——**自动（跟随 Obsidian）/ 中文 / English**；
    面板、命令名与 Notice 全部双语，文案集中于 `src/i18n.ts`。
-   **设置 GUI**：
    -   模板增删 + 行内展开编辑面板（六级 × 六列 + 实时预览 + 起始 / 结束层级 + 跳级占位 +
        白名单编辑器：chip + 匹配方式下拉 + 命中数角标 + 含子标题 ⚠ + 当前文件实时命中预览）。
    -   **路径规则表打磨**（0.6.5）：列表可纵向滚动（表头吸顶）、**分层路径补全**（输入 `/` 逐层展开）、
        每行「✕ 清空此路径」按钮、删除键改为纯文本 ✕。
    -   **防抖延迟滑块**（50–2000ms，带重置按钮）：精确控制自动触发的等待时长。
    -   **「清除全库编号」按钮**（危险区域，**默认折叠**，0.6.5）：一次性剥离全库所有 Markdown 文件的编号前缀，需二次确认。
-   **保留正文起头数字（方案A，0.6.6）**：触发标题编号时**只剥插件自己写的编号**（前缀末尾带不可见的
    Word Joiner 标记），用户正文里像编号的文字（如「2024 年度总结」「API 设计」）**绝不会被误当编号吃掉**，
    首次编号即得 `1 2024 年度总结`、幂等保留。
-   **命令面板**：
    -   「切换自动编号（全局）」「立即重新编号（当前文件）」
    -   **「清除当前文件编号」**（0.6.0）：移除当前文件**所有**编号前缀，还原裸标题，可撤销。
    -   **「清理非本插件的标题编号」**（0.6.6）：只剥**手写 / 导入**的外来编号（覆盖 `1.`/`1)`/`(1)`/`第3章`/
        `一、`/字母/罗马等惯例），**保留**插件自己写的编号——用于导入带手写编号的文档后让插件接管。
-   **双层开关**：「启用插件」（Obsidian 总开关）与「全局自动编号」（面板开关 ↔ 全局命令双向同步）解耦；
    `obsidian-auto-headings: true/false` 单文件 frontmatter 复选框开关——`true`（勾选）文件级**强制**自动编号（覆盖全局关）、
    `false`（未勾选）文件级强制关闭；「立即重新编号」**手动命令绕过**全局开关与 `false`，仅受「能否命中模板」约束。
-   **按路径配置 GUI**：可视化路径规则表（增删、拖拽排序、模板下拉、路径补全、「无根规则」兜底提示条）；
    删除被规则引用的模板时弹「知情确认 + 安全降级」对话框（降级「默认」/ 改投他模板 / 连规则一并删）。
-   代码块内的 `#` 行不误判为标题。
